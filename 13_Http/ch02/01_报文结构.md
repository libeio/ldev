
用于 HTTP 协议交互的信息称为 HTTP 报文。请求端的 HTTP 报文叫做请求报文，响应端的叫做响应报文。

HTTP 报文本身是由多行(CR+LF 作为换行符)数据构成的字符串文本。其大致可分为报文首部和报文主体两块，两者由最初出现的空行(CR+LF)来划分。报文主体不一定是必需的。

请求报文结构如下:
```shell
    +-------------+                        +-------------+
    |   报文首部   |     报文首部内容 ->     |    请求行   |
    +-------------+                        +-------------+
    |   CR + LF   |                        | 请求首部字段 |
    +-------------+                        +-------------+
    |             |                        | 通用首部字段 |
    |   报文主体   |                        +-------------+
    |             |                        | 实体首部字段 |
    +-------------+                        +-------------+
                                           |     其他    |
                                           +-------------+
```

响应报文结构如下:
```shell
    +-------------+                        +-------------+
    |   报文首部   |     报文首部内容 ->     |    状态行   |
    +-------------+                        +-------------+
    |   CR + LF   |                        | 响应首部字段 |
    +-------------+                        +-------------+
    |             |                        | 通用首部字段 |
    |   报文主体   |                        +-------------+
    |             |                        | 实体首部字段 |
    +-------------+                        +-------------+
                                           |     其他    |
                                           +-------------+
```

说明:
* 请求行: 包含用于请求的方法，请示 URI 和 HTTP 版本。
* 状态行: 包含表明响应结果的状态码，原因短语和 HTTP 版本。
* 首部字段: 包含表示请求和响应的各种条件和属性的名类首部。一般有 4 种首部，分别是: 通用首部、请求首部、响应首部和实体首部。
* 其他: 可能包含 HTTP 的 RFC 里未定义的首部(Cookie 等)。
